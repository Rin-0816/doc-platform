## 1. 目的

本システムは、社内で利用する学習資料、技術資料、運用資料を一元管理するための Web ベースのドキュメント管理基盤を提供する。

特に、以下を満たすことを目的とする。

- Markdown を中心に、画像と Mermaid 図を含む文書を扱える
- 文書の閲覧、編集、コメント、検索、履歴管理を一体で提供する
- メインプログラムとプラグインを分離し、用途ごとの拡張を容易にする
- 互換性チェック機能と機能テスト基盤を備え、継続的に安全に拡張できる

## 2. 背景

社内で扱う ICT 学習資料や技術資料は、対象者、前提知識、使用機材、対応バージョンなどの情報を伴うことが多い。

一方で、文書管理そのものに必要な機能と、教材や業務領域ごとに必要な追加機能は性質が異なる。  
そのため、共通機能はメインプログラムに集約し、用途別機能はプラグインとして追加できる構成が望ましい。

## 3. システム方針

### 3.1 基本方針

- メインプログラムは用途非依存の共通機能を提供する
- 用途依存の機能はプラグインとして追加する
- プラグインは明示的な契約に基づいて読み込む
- 互換性チェックに合格しないプラグインは有効化しない
- 主要機能は自動テスト可能な構造にする
- 仕様、設計、運用に関する文書を体系的に整備する

### 3.2 想定利用環境

- 社内ネットワーク上で利用する
- 利用者は各自の PC から Web ブラウザでアクセスする
- サーバー側は標準 Python を中心に構成する
- フロントエンドは必要に応じて CDN 配信のライブラリを利用する

## 4. 利用者と権限

| 役割 | 概要 |
| --- | --- |
| 閲覧者 | 文書の閲覧、検索、コメント閲覧を行う |
| 編集者 | 文書の作成、編集、画像追加、Mermaid 編集、コメント対応を行う |
| 管理者 | ユーザー管理、プラグイン管理、互換性確認、運用設定を行う |

## 5. 対象範囲

### 5.1 対象とする機能

- 文書作成、編集、閲覧
- Markdown、画像、Mermaid 図の表示
- セクション、タグ、カテゴリ、レッスンによる整理
- 用語専用ページ
- 全文検索と絞り込み検索
- 文章、画像、Mermaid 図へのコメント
- 更新履歴、差分表示、過去版保持、復元
- ロールベースの権限制御
- プラグイン管理
- 互換性チェック
- 単体、結合、機能、互換性テスト

### 5.2 初期対象外

- 社外公開
- 高度なワークフロー承認
- 多段階レビュー承認
- 外部 SaaS 連携
- モバイル専用アプリ

## 10. 制約

- サーバー側は標準 Python を基本とする
- フロントエンド側は CDN ライブラリを利用できる
- 社内ネットワークでの利用を前提とする
- 初期リリースでは外部サービス連携を必須としない
- 将来のプラグイン追加を阻害しない設計とする

## 11. 受入条件

- 文書を作成、編集、閲覧できる
- Markdown、画像、Mermaid 図を表示できる
- タグ、カテゴリ、レッスン、用語で整理できる
- 文章、画像、Mermaid 図にコメントできる
- 更新履歴、差分、復元を利用できる
- 閲覧者、編集者、管理者の権限が機能する
- ICT 学習プラグインを追加し、専用メタデータを扱える
- 互換性チェックで不整合なプラグインを検出できる
- 単体、結合、機能、互換性テストを実行できる
