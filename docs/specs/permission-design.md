# 権限設計書

## 1. 目的

閲覧者、編集者、管理者の操作範囲を定義する。

## 2. ロール

| ロール | 概要 |
| --- | --- |
| viewer | 閲覧中心 |
| editor | 作成、編集、復元 |
| admin | システム管理 |

## 3. 操作権限

| 操作 | viewer | editor | admin |
| --- | --- | --- | --- |
| 文書閲覧 | 可 | 可 | 可 |
| 検索 | 可 | 可 | 可 |
| コメント閲覧 | 可 | 可 | 可 |
| コメント投稿 | 可 | 可 | 可 |
| 文書作成 | 不可 | 可 | 可 |
| 文書編集 | 不可 | 可 | 可 |
| 画像追加 | 不可 | 可 | 可 |
| 履歴復元 | 不可 | 可 | 可 |
| プラグイン管理 | 不可 | 不可 | 可 |
| ユーザー管理 | 不可 | 不可 | 可 |

## 4. UI 方針

- 権限のない操作は表示しない
- API 側でも必ず再判定する
- 権限不足は明示的なエラーとして返す

## 5. テスト観点

- ロール別画面表示
- API 権限拒否
- 復元と管理操作の制限
