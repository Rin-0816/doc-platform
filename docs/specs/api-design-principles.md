# API 設計方針書

## 1. 目的

本書は、UI とバックエンドを疎結合に保つための API 設計方針を定義する。

## 2. 基本方針

- UI は業務データを API 経由でのみ取得、更新する
- API は JSON を基本形式とする
- UI は DB や内部実装に依存しない
- 権限判定は API 側で必ず実施する
- エラーは画面側で扱える一定の形式で返す

## 3. 対象 API 群

| API 群 | 目的 |
| --- | --- |
| Auth API | ログイン状態、権限確認 |
| Document API | 文書、セクション、タグ、カテゴリ、レッスン |
| Attachment API | 画像などの添付 |
| Comment API | 文章、画像、Mermaid 図へのコメント |
| Revision API | 履歴、差分、復元 |
| Glossary API | 用語一覧、用語詳細、関連文書 |
| Search API | 全文検索と絞り込み |
| Plugin API | プラグイン一覧、有効化、無効化、互換性確認 |

## 4. インターフェース原則

- 一覧取得と詳細取得を分ける
- 更新系 API は更新後の正規化済みデータを返す
- 画面で必要な関連情報は、過剰でない範囲で API 応答に含める
- プラグイン固有データは名前空間を分ける
- 互換性チェック結果は UI 表示用と機械判定用の両方を満たす構造にする

## 5. 画面との対応

| 画面 | 主に利用する API |
| --- | --- |
| ホーム | Search API、Document API |
| 文書一覧 | Document API、Search API |
| 文書詳細 | Document API、Comment API、Glossary API |
| 編集 | Document API、Attachment API、Plugin API |
| 履歴 | Revision API |
| 差分 | Revision API |
| 用語 | Glossary API |
| プラグイン管理 | Plugin API |

## 6. 権限方針

| 操作 | 必要権限 |
| --- | --- |
| 閲覧 | 閲覧者以上 |
| コメント閲覧 | 閲覧者以上 |
| コメント投稿 | 認証済み利用者 |
| 文書編集 | 編集者以上 |
| 履歴復元 | 編集者以上 |
| プラグイン管理 | 管理者 |

## 7. エラー方針

- 認証エラー
- 権限エラー
- 入力エラー
- 対象未存在
- 互換性エラー
- サーバー内部エラー

各エラーは、少なくとも `code`、`message`、必要に応じて `details` を持つ。

## 8. 後続文書への引き継ぎ

- 個別の URI、リクエスト、レスポンスは API 設計書で定義する
- コメントアンカー、差分、互換性結果の詳細構造は各詳細仕様書で定義する
