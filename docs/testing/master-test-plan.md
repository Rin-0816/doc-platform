# 総合テスト計画書

## 1. 目的

本書は、設計確認から機能確認までを一体で扱う総合テスト方針を定義する。

## 2. テスト区分

| 区分 | 目的 |
| --- | --- |
| 設計レビュー | 要件と設計の整合確認 |
| 単体テスト | 各機能単位の正しさ確認 |
| 結合テスト | サービス間連携の確認 |
| 機能テスト | 主要ユーザー操作の確認 |
| 互換性テスト | プラグイン適合性の確認 |
| 手動 UI 確認 | 操作性、表示、レスポンシブの確認 |

## 3. 自動テスト対象

- 権限別アクセス制御
- 文書 CRUD
- コメント登録、更新、解決状態
- 履歴保存、差分生成、復元
- 検索と絞り込み
- 用語参照
- プラグイン検出、有効化、無効化
- 互換性チェック判定
- runtime hook 登録と core からの委譲
- runtime 経由の plugin data 読込、保存、復元
- panel host への編集 panel、閲覧 panel 登録
- API の正常系、異常系
- 認証プロバイダ追加
- 検索プロバイダ差し替え
- プラグイン無効化後のデータ保持

## 4. 手動確認対象

- 主要画面の導線
- Markdown、画像、Mermaid 図の表示
- 文章、画像、Mermaid 図へのコメント操作
- 画面上の権限差異
- 差分画面と履歴画面の可読性
- 用語探索のしやすさ
- Chrome、Edge、タブレット、モバイルでの表示
- プラグイン有効時と無効時の作成ツール、ビューアー表示
- 空状態、読込中、エラー状態
- 孤立コメント表示
- 互換性エラー時の管理画面表示

## 5. 受入シナリオ

### 5.1 教材作成

1. 編集者が文書を作成する
2. 画像と Mermaid 図を追加する
3. タグ、カテゴリ、レッスンを設定する
4. ICT 教材メタデータを入力する
5. 保存後に履歴が作成される

### 5.2 閲覧と参照

1. 閲覧者が検索から文書へ到達する
2. 目次を使って移動する
3. 用語ページへ遷移する
4. コメント内容を確認する

### 5.3 更新と復元

1. 編集者が文書を更新する
2. 差分を確認する
3. 過去版を復元する
4. 復元操作も履歴に残る

### 5.4 プラグイン管理

1. 管理者がプラグイン一覧を開く
2. 互換性チェックを実行する
3. 不適合プラグインが検出される
4. 適合プラグインのみ有効化できる

### 5.5 拡張基盤

1. 追加認証プロバイダを登録する
2. 検索プロバイダを差し替える
3. ドキュメント保存時に有効プラグイン runtime へ検証と保存が委譲される
4. 作成ツールとビューアーで有効プラグインの panel が表示される
5. プラグインを無効化する
6. UI、検索拡張、validation は停止し、専用データは保持される
7. 再有効化後に保持済みデータが再表示される

### 5.6 バックアップ復元

1. 日次バックアップを取得する
2. 復元手順を実行する
3. 互換性チェックを再実行する
4. 主要機能を確認する

## 6. テスト観点

| 観点 | 内容 |
| --- | --- |
| 正常系 | 想定操作が完了する |
| 異常系 | 権限不足、入力不備、未存在データを適切に扱う |
| 境界系 | 空文書、大量タグ、長文、画像なし、履歴なし |
| 互換性 | API バージョン、依存関係、DB schema の不一致 |
| 委譲 | core が plugin runtime へ読込、保存、復元、検証を委譲できる |
| 拡張 UI | panel host が plugin 有無に応じて作成ツール、ビューアーを切り替える |
| 表示 | 画面崩れ、長文、狭幅表示 |
| 運用 | バックアップ、復元、ログ監視 |

## 7. 完了条件

- 要件定義と設計文書の対応が確認されている
- 主要 API と主要画面のテスト観点が網羅されている
- 主要ユーザーフローが完走できる
- 不適合プラグインを検出できる
- PC、タブレット、モバイルで致命的な表示崩れがない
- プラグイン無効化後も専用データが保持される
- core が plugin 固有実装を持たず、runtime 委譲のテストで確認できる
- plugin 有無の両条件で作成ツールとビューアーが成立する
- バックアップ復元手順を実施できる
