# 運用手順書

## 1. 目的

日常運用、更新、バックアップ、復元、障害対応を定義する。

## 2. 日常運用

- サービス起動確認
- 最新互換性チェック結果の確認
- エラーログ確認
- 日次バックアップ完了確認

## 3. バックアップ

- 頻度: 日次
- 保持: 複数世代保持
- 対象:
  - SQLite DB
  - 添付ファイル
  - プラグイン設定
  - 設計文書
- 定期的に復元確認を行う

## 4. 更新運用

1. 更新前バックアップ
2. コア更新
3. migration 適用状況確認
4. プラグイン互換性再確認
5. 機能テスト実行
6. ログ確認

## 5. 復元

1. サービス停止
2. DB 復元
3. 添付ファイル復元
4. migration 状態確認
5. 互換性チェック
6. 主要機能確認

## 6. 監視対象

- 起動失敗
- DB 接続失敗
- 添付欠損
- 互換性 `ERROR`
- 認証失敗の急増
- バックアップ失敗

## 7. 障害切り分け

1. サービス状態
2. DB 読込
3. 互換性結果
4. 添付ファイル整合
5. 認証 / 権限設定

## 8. 定期確認

- バックアップ復元試験
- 古い教材の最終確認日
- コメント滞留
- プラグイン更新状況
- ログ肥大化
