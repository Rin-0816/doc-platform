# 作業台帳

## 1. 進め方

本プロジェクトは、設計成果物を段階ゲートで確認しながら進める。

| ゲート | 目的 | 主成果物 |
| --- | --- | --- |
| Gate 1 | 要件を確定する | 要件定義書 |
| Gate 2 | 全体構成と責務分離を確定する | 全体アーキテクチャ設計書、API 設計方針書 |
| Gate 3 | 利用者導線と操作仕様を確定する | 画面仕様書 |
| Gate 4 | 品質確認の方法を確定する | 総合テスト計画書 |
| Gate 5 | 詳細仕様へ展開する | プラグイン仕様書、DB 設計書、各種詳細仕様書 |

## 4. レビュー観点

- 要件 ID と設計成果物が追跡できること
- API と画面の責務が混在していないこと
- プラグイン追加時にコア改修が不要な範囲を明確化できていること
- テスト計画が設計上の主要リスクを覆っていること
- 画面仕様に空状態、エラー状態、権限制御が含まれていること
